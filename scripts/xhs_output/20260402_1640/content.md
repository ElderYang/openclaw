# OpenClaw 探索日记①：为什么我用它替代 5 个 APP

## 我的痛点：每天被重复工作淹没

### 场景 1：每天早上查股票（20 分钟）

**以前的操作**：
```
7:30 闹钟响 → 摸到手机 → 解锁 → 打开同花顺
→ 看外盘：道琼斯 +0.55% 纳指 +1.02%
→ 关掉同花顺 → 打开东方财富
→ 看 A 股：上证 -0.23% 创业板 +0.35%
→ 关掉东方财富 → 打开自选股
→ 看持仓：科创 50ETF +1.25% 半导体 +2.10%
→ 关掉自选股 → 打开雪球
→ 看消息：英伟达发布新芯片...
→ 关掉雪球 → 打开富途
→ 看港股：恒生科技指数...
```

**一共**：5 个 APP，每个点 4-5 下，耗时 20 分钟

**问题**：
- 同样的数据要看 5 遍（怕漏）
- 每个 APP 界面不一样，找数据很烦
- 周末还要手动汇总到 Excel

---

### 场景 2：午休发小红书（1 小时）

**以前的操作**：
```
12:00 吃完饭 → 想发篇小红书
→ 打开 Canva 找模板（15 分钟）
→ 选图片、改文字、调颜色（20 分钟）
→ 导出 5 张图片（5 分钟）
→ 打开小红书 → 写文案（15 分钟）
→ 想标题、加标签、选封面（10 分钟）
→ 点击发布 → 等审核
```

**一共**：找图 15 分钟 + 做图 20 分钟 + 写文案 15 分钟 + 发布 10 分钟 = 1 小时

**问题**：
- 每天都要做同样的事
- 找图最痛苦，好看的模板都用烂了
- 写文案憋半天，不知道写什么

---

### 场景 3：下班整理数据（30 分钟）

**以前的操作**：
```
17:00 下班前 → 领导要日报
→ 打开 Excel 填今天的数据（10 分钟）
→ 打开 PPT 更新图表（15 分钟）
→ 复制粘贴、调整格式、检查错别字（5 分钟）
→ 发到飞书工作群
```

**一共**：30 分钟

**问题**：
- 数据都是现成的，但要手动复制
- 格式经常错，要反复调整
- 同样的 PPT 模板用了 100 遍了

## 直到有一天，我算了笔账

**每天浪费时间**：
- 查股票：20 分钟
- 发小红书：60 分钟
- 整理数据：30 分钟
- **合计**：110 分钟 = 1.8 小时

**每月**：9 × 4 = 36 小时
**每年**：36 × 12 = 432 小时 = **18 天**

> "这些时间拿来写代码、学习、陪家人不香吗？"

## 我开始寻找解决方案

### 尝试 1：Python 脚本（2026 年 2 月）❌

**做了什么**：
```python
import requests
import schedule

def check_stock():
    # 获取股票数据
    # 发送邮件
    pass

schedule.every().day.at("07:30").do(check_stock)
```

**结果**：❌ 失败

**原因**：
- 脚本要一直运行，占用电脑
- 电脑一关机就失效
- 没有推送，还得主动查邮件
- 写推送逻辑太麻烦（SMTP、模板...）

**耗时**：2 天写代码 + 1 天调试 = 放弃

---

### 尝试 2：iOS 快捷指令（2026 年 2 月）⚠️

**做了什么**：
- 创建快捷指令：获取网页数据 → 发送到飞书
- 设置自动化：每天早上 7:30 执行

**结果**：⚠️ 半成功

**问题**：
- 只能获取简单数据（复杂 API 搞不定）
- 无法处理错误（失败了没提示）
- 无法调试（不知道哪步错了）
- 手机必须联网，不然不执行

**坚持了**：1 周 → 放弃

---

### 尝试 3：Zapier / IFTTT（2026 年 3 月）❌

**做了什么**：
- 注册 Zapier（免费版）
- 配置 RSS → 飞书 工作流

**结果**：❌ 失败

**原因**：
- 国内服务支持差（很多 API 连不上）
- 免费版限制太多（每月 100 次任务）
- 无法自定义逻辑（只能简单触发）
- 延迟高（有时候下午才收到早上的消息）

**耗时**：1 天配置 + 2 天调试 = 放弃

---

## 最后我遇到了 OpenClaw

### 为什么选它

**核心优势**：
1. ✅ **本地运行** - 数据在自己手里，不怕泄露
2. ✅ **Python 脚本** - 逻辑随意写，想多复杂都行
3. ✅ **launchd 定时** - macOS 系统级，关机也能执行（唤醒后）
4. ✅ **飞书原生** - 推送稳定，支持卡片消息
5. ✅ **开源** - 可以改代码，有问题能自己修

**第一周进展**：
```
Day 1（3/28）：环境搭建 ✅
  - 安装 Node.js（20 分钟）
  - 安装 OpenClaw（10 分钟）
  - 配置飞书集成（30 分钟，卡在 API Key）
  - 踩坑：网络问题，翻墙才装上

Day 2（3/29）：第一个定时任务 ✅
  - 写天气预报脚本（30 分钟）
  - 配置 launchd（20 分钟）
  - 早上 7 点准时推送，成功！

Day 3（3/30）：股市早报 ✅
  - 集成 QVeris API（1 小时）
  - 多数据源降级策略（2 小时）
  - 失败自动重试（30 分钟）
  - 早上收到推送，数据完整！

Day 4（3/31）：内容创作自动化 🔄
  - 尝试小红书自动发布（2 小时）
  - 失败：UI 变化太快，选择器失效
  - 改为半自动：AI 生成内容，人工发布
  - 从 1 小时缩短到 5 分钟！

Day 5（4/1）：反思和调整 ✅
  - 接受"全自动不可行"
  - 建立内容规划系统
  - 决定记录真实过程，不画饼
```

## 真实感受：爽和不爽都有

### 爽的点

**1. 数据终于统一了**
```
以前：5 个 APP 来回切，怕漏信息
现在：飞书一条消息，外盘+A 股 + 持仓 + 消息 全都有
```

**2. 早上多睡 20 分钟**
```
以前：7:10 起床，迷迷糊糊查股票
现在：7:30 起床，洗漱完看飞书推送
```

**3. 内容创作变快了**
```
以前：找图 + 写文案 + 发布 = 1 小时
现在：AI 生成内容 + 图片 = 5 分钟，我只负责发
```

**4. 有成就感**
```
看着自己搭建的系统每天自动运行
就像有个小助手在帮我干活
```

---

### 不爽的点

**1. 配置过程确实麻烦**
- 不是"开箱即用"
- 需要懂一点技术（terminal、JSON、Python）
- 网络问题要自己解决（翻墙、代理）

**2. 自动化不是万能的**
- 小红书自动发布失败了（UI 变化太快）
- 有些平台反爬太严（验证码、IP 限制）
- 接受"半自动"更务实

**3. 维护成本被低估了**
- API 会失效（Key 过期、接口变更）
- UI 会变化（选择器失效）
- 要持续跟进修复（每周花 1-2 小时）

---

## 给想尝试的人

### 适合谁

✅ 有一定技术基础（会用 terminal、懂 JSON）
✅ 愿意折腾配置（看文档、查错误、调试）
✅ 有明确的自动化需求（重复工作多）
✅ 能接受"不完美"（有些功能会失败）

### 不适合谁

❌ 想要"一键搞定"（这不是商业软件）
❌ 完全不懂技术（terminal 是什么都不知道）
❌ 期望 100% 自动化（有些就是做不到）
❌ 不想维护（配置完就不管了）
