## ✅ 2026-04-01 15:42 - 重复回复问题 - 最终根因与修复

### 问题描述
飞书 webhook 在 2 秒内推送 3 次相同消息，每次 `message_id` 不同，导致 Gateway 回复 3 条相同内容。

### 最终根因（深入排查后确认）
**飞书 webhook 推送机制问题**：
1. 飞书在 2 秒内推送 3 次相同的用户消息
2. 每次推送的 `message_id` 不同（飞书生成不同的 ID）
3. Gateway 的去重逻辑基于 `message_id`，无法识别这是同一条消息
4. 导致 Gateway 处理 3 次，回复 3 条相同内容

### 去重失效分析
**为什么 `dedup: {enabled: true, windowMs: 5000}` 不生效？**
- Gateway 的 `processingClaims` 去重基于 `message_id`
- 飞书推送的 3 次消息 `message_id` 不同
- 去重逻辑认为这是 3 条不同的消息
- 结果：3 次都处理，3 次都回复

### 最终修复方案
**添加 `inboundDebounceMs` 配置**：
```json
"channels": {
  "feishu": {
    "inboundDebounceMs": 2000,  // 2 秒内相同聊天/发送者的消息合并
    "accounts": {
      "main": {
        "streaming": true,
        "renderMode": "card",
        "inboundDebounceMs": 2000,  // 账号层级也配置
        "dedup": {"enabled": true, "windowMs": 5000}
      }
    }
  }
}
```

**工作原理**：
- Gateway 的 `inboundDebouncer` 基于 `chat_id + sender_id + root_id` 生成去重 key
- 2 秒内相同 key 的消息会被合并，只处理第一次
- 这样即使飞书推送多次，Gateway 也只处理 1 次

- **日志文件**：`/tmp/openclaw/openclaw-2026-04-01.log`
