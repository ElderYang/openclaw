```python
# 美股：Tushare + Yahoo Finance
# A 股：AkShare + Tushare + 东方财富
```

### 校验规则
1. 双源一致 (误差<0.5%) → ✅ 标注
2. 单源数据 → ⚠️ 标注
3. 数据延迟 → ⚠️ 标注

2. Fallback 到东方财富 API，但**只获取了点位字段 `f43`，没有获取涨跌幅字段**
3. 当 Tushare 失败时，涨跌幅字段为 `None`，显示为 **0%**

### 修复方案（v21.3 → v21.4）
**Commit**: `5204922`

**修改前**（❌ 手动计算）：
```python
fields=f43,f170  # 点位 + 昨收
if d.get('f170') and d['f170'] != 0:
    em_pct = ((d.get('f43', 0) - d.get('f170', 0)) / d.get('f170', 1)) * 100
    ts_pct = round(em_pct, 2)
```

**修改后**（✅ 直接获取）：
```python
fields=f43,f49   # 点位 + 涨跌幅 (%)
if d.get('f49') is not None:
    em_pct = float(d.get('f49', 0))
    ts_pct = em_pct
```

| 字段 | 含义 |
|------|------|
| `f43` | 当前点位 |
| `f49` | **涨跌幅 (%)** ← 直接用这个！ |
| `f50` | 涨跌额 |
| `f170` | 昨收 |

### 优化收益
- 代码简洁度：⬆️ 50%（4 行 → 2 行）
- 计算误差风险：⬇️ 100%（无需除法运算）
- 可维护性：⬆️ 高（一目了然）

### 教训
- 优先使用 API 官方字段，避免手动计算
- 手动计算增加误差风险和代码复杂度
- 仔细阅读 API 文档，了解所有可用字段
