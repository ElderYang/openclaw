## [LRN-20260316-004] 多源校验是必要的

**类别**: 数据准确性  
**严重程度**: 🔴 高

### 原则
- 绝不依赖单一数据源
- 至少 2 个数据源相互印证
- 数据不一致时标注⚠️

### 实践
```python
# 美股：Tushare + Yahoo Finance
```

### 校验规则
1. 双源一致 (误差<0.5%) → ✅ 标注
2. 单源数据 → ⚠️ 标注
3. 数据延迟 → ⚠️ 标注
4. 获取失败 → "待确认"

## [LRN-20260329-001] 小红书 MCP 服务器图片上传检测 bug

**类别**: 技术配置 / Bug  
**严重程度**: 🟡 中  
**触发时间**: 2026-03-29 14:51

### 问题描述
MCP 服务器检测的是"图片提交数量"完成（5/5），但小红书网页还需要时间处理图片（压缩、转码）。

**表现**:
- MCP 显示"图片上传完成"
- 小红书网页提示"图片上传中，请稍后"
- MCP 直接点击发布，导致失败

### 根因分析
1. MCP 只检测提交请求完成
2. 未等待小红书网页端处理完成
3. 缺少"图片上传中"提示消失的等待逻辑

### 解决方案
需要 MCP 开发者修复：
```javascript
// 添加等待逻辑
await page.waitForSelector('text=图片上传中，请稍后', { state: 'detected' });
await page.waitForSelector('text=图片上传中，请稍后', { state: 'detached', timeout: 30000 });
// 然后再点击发布按钮
```

### 临时规避
- 手动发布时，等待网页"图片上传中"提示消失后再点击发布
- 检查小红书 APP 确认发布成功
