- 标题：`input[placeholder*="标题"]`、`input[class*="title"]`、`.publish-page-title input`
- 正文：`div[contenteditable="true"]`、`textarea[placeholder*="正文"]`、`.ql-editor[contenteditable]`
- 图片上传：`input[type="file"][accept*=".jpg"]`、`.upload-area`

**尝试过的 URL**：
- `https://creator.xiaohongshu.com/publish/publish?source=official`
- `https://creator.xiaohongshu.com/publish/publish`（移除 query 参数）

**调试文件**：`/tmp/xhs_*.png`、`/tmp/xhs_page_source.html`

## 📈 复盘报告指数涨跌幅修复（2026-04-02 17:16-17:21）

### 问题

**Yahoo Finance 的 `previousClose` 字段返回错误日期**：
- 返回的是 03-31 收盘价（3891.86）
- 但正确的昨收应该是 04-01 收盘价（3948.55）

**正确计算**：
- (3919.29 - 3948.55) / 3948.55 = -0.74% ✅

### 修复方案

**混合数据源策略**：
```python
# Yahoo Finance：获取今天收盘价
close = float(hist['Close'].iloc[-1])

# AkShare：获取昨收价（历史数据）
df = ak.stock_zh_index_daily(symbol="sh000001")
yesterday_close = df['close'].iloc[-2]  # -2 是昨收，-1 是今天收盘

# 手动计算涨跌幅
pct_chg = ((close - yesterday_close) / yesterday_close) * 100
```

## 📈 指数涨跌幅修复最终版（2026-04-02 18:00）

### 问题复现
**现象**：修复后深证成指和创业板指仍然显示错误涨跌幅

**测试数据**：
- 上证指数：3919.29，**+0.00%** ❌（应为 -0.74%）
- 深证成指：13486.94，**+0.07%** ❌（应为 -1.60%）
- 创业板指：3172.65，**-0.39%** ❌（应为 -2.31%）
- 科创 50：1262.18，**-0.00%** ❌（应为 -2.77%）

### 根因分析（第 3 层问题）

**第 1 层**：Yahoo Finance `previousClose` 不稳定
- 有时返回昨天，有时返回前天
- 之前正确是运气好 🎲

**第 2 层**：AkShare 数据更新不一致
- 上证指数：当天更新（最后一行 = 今天）
- 深证/创业板：次日更新（最后一行 = 昨天）
- 代码假设所有指数都是次日更新 ❌

**第 3 层**：日期类型比较失败
- AkShare 返回 `datetime.date` 类型
- 代码比较 `datetime.date == str` → 永远 False ❌

### 最终修复方案

```python
# 根据日期动态判断昨收
last_date = ak_df.iloc[-1]['date']
today_str = datetime.now().strftime("%Y-%m-%d")

if str(last_date) == today_str:
    # 最后一行是今天，用倒数第二行（昨收）
    prev_close = ak_df.iloc[-2]['close']
else:
    # 最后一行是昨天，用最后一行（昨收）
    prev_close = ak_df.iloc[-1]['close']

# 计算涨跌幅
pct_chg = ((yahoo_close - prev_close) / prev_close) * 100
```
