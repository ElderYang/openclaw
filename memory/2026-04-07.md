## 🔧 飞书插件故障排查（2026-04-07 19:00-19:37）

### 问题现象
1. 飞书渠道消息在 WebUI 不显示
2. 飞书工具调用返回 400 错误
3. 错误代码：`99992356 - Invalid ids: [user:ou_a040d98b29a237916317887806d655de]`

### 根因分析

**问题 1：工具被禁用**
- `tools.allow: []` 空数组导致飞书工具完全被禁用
- 即使 `profile: "full"` 也无法使用工具

**问题 2：插件配置重复**
- `plugins.allow: ['@openclaw/feishu', 'feishu']` 重复配置
- `plugins.entries` 中存在过期引用

**问题 3：会话渠道隔离（非 bug）**
- 飞书和 WebUI 是**两个独立的会话渠道**
- 会话 Key 不同：
  - 飞书：`agent:main:feishu:direct:ou_a040d98b29a237916317887806d655de`
  - WebUI：`agent:main:main`
- 消息历史**不互通**是 OpenClaw 的设计行为

### 修复方案

1. **清理 plugins.allow**：`['@openclaw/feishu', 'feishu']` → `['@openclaw/feishu']`
2. **删除 tools.allow 空数组**：让系统使用 `full` profile 自动选择工具
3. **删除重复的 plugins.entries**：只保留 `@openclaw/feishu`
4. **删除 main 账号的独立 model 配置**：使用默认模型

### 验证结果

```bash
✅ feishu_chat info - 成功返回会话信息
✅ feishu_chat members - 成功返回成员列表
```

### 关键学习

1. **tools.allow: [] 会禁用所有工具** - 即使 profile 设置为 full
2. **多渠道会话隔离是设计行为** - 不是 bug，无需修复
3. **飞书 DM 场景使用 chat_id** - `oc_...` 格式配合 `chat_id` 类型
4. **配置清理重要性** - 过期插件引用会导致警告

## 🔧 飞书插件 ID 不匹配问题（2026-04-07 19:48 最终修复）

### 问题现象
修复后工具调用仍然返回 400 错误，Gateway 持续报错：
```
plugin not found: @openclaw/feishu
```

### 根本原因

**插件 ID 不匹配**：
- 飞书插件 `openclaw.plugin.json` 定义的 ID：`"feishu"`
- 配置文件 `openclaw.json` 中使用的 ID：`"@openclaw/feishu"`
- **ID 不一致导致插件无法加载**

### 最终修复

```json
// 修复前（错误）
"plugins": {
  "allow": ["@openclaw/feishu", "feishu"],
  "entries": {
    "@openclaw/feishu": {...},
    "feishu": {...}
  }
}

// 修复后（正确）
"plugins": {
  "allow": ["feishu"],
  "entries": {
    "feishu": {
      "enabled": true,
      "config": {}
    }
  }
}
```

### 验证结果


### 关键学习

1. **插件 ID 必须匹配** - `openclaw.plugin.json` 中的 `id` 字段必须与 `openclaw.json` 中的 `plugins.allow` 和 `plugins.entries` 完全一致
2. **npm 包名 ≠ 插件 ID** - 即使 npm 包名是 `@openclaw/feishu`，插件 ID 可能是 `feishu`
3. **检查插件 ID 的方法**：
   ```bash
   cat ~/.openclaw/extensions/node_modules/@openclaw/feishu/openclaw.plugin.json | grep '"id"'
   ```

## 🔧 飞书工具 chat_id 格式问题（2026-04-07 19:50 最终定位）

### 问题现象
配置修复后工具调用仍然返回 400 错误：
```
code: 99992356
msg: Invalid ids: [user:ou_a040d98b29a237916317887806d655de]
```

### 排查过程

| 步骤 | 假设 | 验证结果 |
|------|------|----------|
| 1 | tools.allow: [] 禁用工具 | ❌ 修复后仍失败 |
| 2 | 插件 ID 不匹配 | ❌ 修复后仍失败 |
| 3 | appId/appSecret 错误 | ❌ 直接调用 API 成功 |
| 4 | **chat_id 格式错误** | ✅ **找到根因** |

### 根本原因

**chat_id 参数使用了错误的 ID 格式**：

```json
// ❌ 错误（工具调用实际使用的）
{"action": "info", "chat_id": "user:ou_a040d98b29a237916317887806d655de"}

// ✅ 正确（应该使用的）
{"action": "info", "chat_id": "oc_1cf4deaeb014ea9b1ce5cd7c8a35fad8"}
```

### 飞书 ID 格式说明

| 类型 | 前缀 | 示例 | 用途 |
|------|------|------|------|
| **open_id** | `ou_` | `ou_a040d98b29a237916317887806d655de` | 用户身份标识 |
| **chat_id** | `oc_` | `oc_1cf4deaeb014ea9b1ce5cd7c8a35fad8` | 会话标识 ← **工具调用需要这个** |
| user_id | `cli_` | `cli_xxx` | 企业内部用户 ID |

### 关键学习

1. **飞书工具调用必须使用 chat_id（oc_开头）**，不能用 open_id（ou_开头）
2. **open_id 用于标识用户身份**，chat_id 用于标识会话
3. **DM 私聊场景**：每个用户与机器人的私聊有独立的 chat_id
4. **直接使用飞书 SDK 调用 API 可以验证配置正确性**

### 验证方法

```bash

curl -X POST "https://open.feishu.cn/open-apis/im/v1/chats/oc_1cf4deaeb014ea9b1ce5cd7c8a35fad8" \
  -H "Authorization: Bearer <tenant_access_token>"
```
