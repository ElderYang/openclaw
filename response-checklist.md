# 响应前检查清单（每次回复前必看）

## ✅ 任务追踪检查
- [ ] 是否有后台任务在执行？→ 检查 process 状态
- [ ] 是否有任务已完成但未通知？→ 检查 task-tracker.json
- [ ] 是否有用户问题未回答？→ 检查对话历史

## ✅ 反馈规范
- [ ] 长任务是否告知了预计时间？
- [ ] 超过 2 分钟的任务是否汇报了进度？
- [ ] 任务完成是否主动通知了？
- [ ] 任务失败是否说明了原因 + 建议？

## ✅ 问题回答
- [ ] 用户的问题是否都回答了？
- [ ] 是否有遗漏的问题？
- [ ] 回答是否清晰完整？

## ✅ 下一步行动
- [ ] 是否告知了下一步做什么？
- [ ] 是否需要用户确认？
- [ ] 是否有待办事项？

---

## 🚨 常见错误（避免再犯）

1. ❌ 用户问了问题，去查了但没回复结果
2. ❌ 后台任务执行中，没有进度汇报
3. ❌ 任务完成了，没有主动通知
4. ❌ 任务失败了，没有说明原因
5. ❌ 多个问题，只回答了部分

---

## 📝 使用方式

**每次回复前**：
1. 快速浏览此清单（10 秒）
2. 确认没有遗漏
3. 再发送回复

**每次长任务**：
1. 开始时告知预计时间
2. 每 1-2 分钟汇报进度
3. 完成/失败立即通知
